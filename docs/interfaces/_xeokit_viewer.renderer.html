Interface Renderer

Manages storage and rendering of meshes for objects in a Viewer.

Used by a Viewer internally, to allocate and render geometry and materials using a browser 3D graphics API (eg. WebGL, WebGPU).

Usage

 
import {Viewer} from "@xeokit/viewer";

const myViewer = new Viewer({
    id: "myViewer",
    renderer: new MyRenderer({ })
});

createModel

  • createModel(params: ViewerModelParams): ViewerModel
    • Defined in viewer/src/Renderer.ts:77

    Returns a new ViewerModel that will be stored and rendered by this Renderer.

    The ViewerModel provides an interface through which we can then build geometry and materials within it. Once we've built the ViewerModel and called ViewerModel.build, the Renderer will immediately begin rendering it all Views that we registered previously with Renderer.registerView.

    When we've finished with the ViewerModel, we then call ViewerModel.destroy to destroy it.

    Parameters

    • params: ViewerModelParams

      ViewerModel creation params

    Returns ViewerModel
