Interface Renderer

Pluggable, internal rendering strategy for a Viewer.

Used by a Viewer internally, to allocate and render geometry and materials using a browser 3D graphics API (eg. WebGL, WebGPU).

You don't normally need to know about this interface, unless you're configuring your Viewer with a custom strategy.

A Viewer has a WebGLRenderer by default.

Usage

 
import {Viewer} from "https://cdn.jsdelivr.net/npm/@xeokit/xeokit-viewer/dist/xeokit-viewer.es.min.js";

const myViewer = new Viewer({
    id: "myViewer",
    renderer: new MyRenderer({ })
});

createModel

  • createModel(params: SceneModelParams): SceneModel
    • Defined in viewer/scene/Renderer.ts:80

    Returns a new SceneModel that will be stored and rendered by this Renderer.

    The SceneModel provides an interface through which we can then build geometry and materials within it. Once we've built the SceneModel and called SceneModel.finalize, the Renderer will immediately begin rendering it all Views that we registered previously with Renderer.registerView.

    When we've finished with the SceneModel, we then call SceneModel.destroy to destroy it.

    Parameters

    • params: SceneModelParams

      SceneModel creation params
