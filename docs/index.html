xeokit-viewer

@xeokit

Graphics SDK for browser-based AECO visualization.

Features
  • Next-generation browser-based 3D/2D viewer from @xeolabs
  • For BIM & AEC applications
  • Fast rendering
  • Compact memory footprint
  • Multiple canvases
  • Semantic ER data model
  • Pluggable graphics (WebGL, WebGPU..)
  • Natively TypeScript

Getting started

To learn how @xeokit works, see Concepts. To get started developing with the SDK, see SDK Installation below. Find Functions for example scripts created with the SDK already. To use the commandline interface, see Commandline (CLI). If you're interested in contributing to or customizing the project, see contributing.

SDK Installation

Install the core WebViewer for programmatic use:

npm install --save @xeokit/webviewer

Then, import some modules:

// ES Modules.
import {WebViewer} from '@xeokit/webviewer';

// CommonJS.
const {WebViewer} = require('@xeokit/webviewer');

Concepts

The WebViewer class is the core component of @xeokit. The WebViewer has the following main components:

  • A Scene containing SceneModels and SceneObjects, which define the geometry and materials of our models.
  • A Data containing DataModels and DataObjects, which describe the semantics and structure of our models.
  • One or more Views, that each create an independent view of the Scene. Each View has its own canvas, Camera, and ViewObjects, which define the appearance of each SceneObject in that particular View.
