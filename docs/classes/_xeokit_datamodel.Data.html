Class Data

An entity-relationship data model.

Features
  • Generic entity-relationship graph
  • Supports multiple, federated data models
  • Builder API to prgrammatically create data models
  • Supports any schema that's expressable as an ER graph
  • Supports IFC models
  • Graph traversals to query objects

Quickstart
  • Create @xeokit/datamodel/DataModel|DataModels with createModel
  • Create DataObjects with @xeokit/datamodel/DataModel.createObject
  • Create PropertySets with @xeokit/datamodel/DataModel.createPropertySet
  • Create Relationships with @xeokit/datamodel/DataModel.createRelationship
  • Query with searchDataObjects
  • When built, be sure to finalize each DataModel with @xeokit/datamodel/DataModel.build
  • When no longer needed, be sure to destroy each DataModel with @xeokit/datamodel/DataModel.destroy

Example 3. Querying DataObjects

In our third example, we'll extend our previous example to use the searchDataObjects method to traverse our data graph and fetch the IDs of the DataObjects we find on that path.

One example of where we use this method is to query the aggregation hierarchy of the DataObjects for building a tree view of an IFC element hierarchy.

const objectIds = [];

myData.searchDataObjects({
    startObjectId: "table",
    includeObjects: [mySchema.FURNITURE_TYPE],
    includeRelated: [mySchema.AGGREGATES_REL],
    objectIds
});

// objectIds == ["table", "tableTop", "redLeg", "greenLeg", "blueLeg", "yellowLeg"];

view.setObjectsHighlighted(objectIds, true);
