Class Viewer

The browser-based viewer at the core of @xeokit/xeokit-viewer.

Overview

A Viewer has three main components:

  • Scene - Contains geometry and materials for models.
  • Data - Contains optional semantic information about the models.
  • Views - One or more independently-configurable views of the models, each with its own canvas.

A Viewer can also be configured with a custom Renderer strategy, in case we need to customize the way the Viewer allocates and renders 3D models on the underlying browser API (eg. WebGL, WebGPU).

Usage

Create a viewer:

import {Viewer, constants} from "https://cdn.jsdelivr.net/npm/@xeokit/xeokit-viewer/dist/xeokit-viewer.es.min.js";

const myViewer = new Viewer({
    id: "myViewer"
});

Create model geometry and materials:

const mySceneModel = myViewer.scene.createModel({
    id: "myModel"
});

mySceneModel.createGeometry({
    id: "myGeometry",
    primitive: constants.TrianglesPrimitive,
    positions: [...],
    indices: [...]
    //...
});

mySceneModel.createMesh({
    id: "myMesh",
    geometryId: "myGeometry",
    //...
});

mySceneModel.createObject({
    id: "myObject",
    meshIds: ["myMesh"],
    viewLayer: "main"
    //...
});

myModel.finalize();

Define optional semantic information for the model:

const myDataModel = myViewer.data.createModel({
    id: "myModel"
});

myDataModel.createObject({
    id: "myObject",
    name: "Some object",
    type: "MyType"
});

Create a view of the model:

const view1 = myViewer.createView({
    id: "myView",
    canvasId: "myCanvas1"
});

view1.camera.eye = [-3.933, 2.855, 27.018];
view1.camera.look = [4.400, 3.724, 8.899];
view1.camera.up = [-0.018, 0.999, 0.039];
view1.camera.projection = "perspective";
view1.cameraControl.navMode = "orbit";

Customize the view:

view1.viewObjects["myObject"].highlighted = true;

        The Real-space 3D origin, in current measurement units, at which the World-space coordinate origin [0,0,0] sits.

createView

  • createView(params: ViewParams): View
    • Defined in viewer/Viewer.ts:261

    Creates a new View within this Viewer.

    • The maximum number of views you're allowed to create is provided in ViewerCapabilities.maxViews. This will be determined by the Renderer implementation the Viewer is configured with.
    • Fires a "viewCreated" event on this Viewer.
    • To destroy the View after use, call View.destroy, which fires a "viewDestroyed" event on this Viewer.
    • You must add a View to the Viewer before you can create or load content into the Viewer's Scene.

    Usage

     
    const view1 = myViewer.createView({
         id: "myView",
         canvasId: "myCanvas1"
     });
