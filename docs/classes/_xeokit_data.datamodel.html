Class DataModel

A buildable entity-relationship semantic data model.

See Data for usage examples.

Summary
  • A DataModel is a generic entity-relationship graph of DataObjects, PropertySets and Relationships
  • Can be used for IFC and all other schemas that are expressable as an ER graph
  • Created with Data.createModel
  • Stored in Data.models

createObject

  • createObject(dataObjectParams: DataObjectParams): DataObject
    • Defined in data/src/DataModel.ts:224

    Creates a DataObject in this DataModel.

    Each DataObject has a globally-unique ID in DataObject.id, with which it's registered in Data.objects and DataModel.objects.

    If DataObjectParams.id matches a DataObject that already exists (ie. already created for a different DataModel), then this method will reuse that DataObject for this DataModel, and will ignore any other DataObjectParams parameters that we provide. This makes the assumption that each value of DataObjectParams.id is associated with a single value for DataObjectParams.type and DataObjectParams.name. This aligns well with IFC, in which wewe never have two elements with the same ID but different types or names.

    Each DataObject automatically gets destroyed whenever all the DataModels that share it have been destroyed.

    We can attach our DataObject as child of an existing parent DataObject. To do that, we provide the ID of the parent in {@link DataObjectParams.parentId}. Following the reuse mechanism just described, the parent is allowed to be a DataObject that we created in a different DataModel. If the parent does not exist yet, then our new DataObject will automatically become the parent's child when the parent is created later.

    Parameters

    • dataObjectParams: DataObjectParams

    Returns DataObject
