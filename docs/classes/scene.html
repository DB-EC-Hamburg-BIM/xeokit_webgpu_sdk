Class Scene

Contains geometric representations of the models and objects within a Viewer.

Summary
  • Located at Viewer.scene
  • Stores SceneModels in Scene.models
  • Stores SceneObjects in Scene.objects
  • Provides the center and axis-aligned World-space boundary of all SceneObjects in Scene.center and Scene.aabb, respectively
  • Use Scene.createModel to create SceneModels
  • Use SceneModel.createObject to create SceneObjects
  • Views automatically contain ViewObjects to represent all existing SceneObjects

Accessors

aabb

  • get aabb(): Float64Array
    • Defined in viewer/scene/Scene.ts:93

    Gets the World-space axis-aligned 3D boundary (AABB) of this Scene.

    The AABB encompases the boundaries of all SceneModel s currently in Scene.models, and is represented by a six-element Float64Array containing the min/max extents of the axis-aligned volume, ie. [xmin, ymin,zmin,xmax,ymax, zmax].

    When the Scene has no content, the boundary will be an inverted shape, ie. [-100,-100,-100,100,100,100].

    Returns Float64Array

createModel

  • createModel(params: SceneModelParams): SceneModel
    • Defined in viewer/scene/Scene.ts:177

    Creates a new SceneModel within this Scene.

    The SceneModel represents a new model within the Scene and provides an interface through which we can then build geometry and materials within it.

    When we've finished building our SceneModel, we then call SceneModel.finalize, which causes it to immediately begin rendering within all the Views we created previously with Viewer.createView.

    As that happens, each View automatically gets a ViewObject for each of the SceneModel's SceneObjects, to independently represent that SceneObject's visual state in that View.

    When we've finished with the SceneModel, we destroy it using SceneModel.destroy. That will automatically remove its ViewObjects from all our existing Views.

    Usage

     
    const mySceneModel = myViewer.scene.createModel({
   id: "myModel"
});
